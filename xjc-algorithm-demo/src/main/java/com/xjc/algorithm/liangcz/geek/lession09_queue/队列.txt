1、队列也栈一样，是一种操作受限的数据结构
队列在底层架构起了很大的作用，如阻塞队列、循环队列、并发队列等
2、和栈一样，队列可以用数组实现，也可以用链表实现。分为顺序队列和链式队列。
3、 循环队列避免出列和入列对数据的搬运
    要注意数据满和数据为空的判断
   为了区分数据满和数据空，增加了一个位置的空
4、循环队列解决了数据出入列对数据的搬运造成的时间复杂度
5、阻塞队列，当队列满时候，从队列插入会阻塞。当队列为空时候，从队列取数会被阻塞。
   阻塞队列可以用于生产-消费者消息模型
6、并发队列指的是线程安全队列，在队列上加锁可以提高线程安全。但是锁的颗粒度大，会同时限制出列和入列。
顺序队列，加CAS可以提供高性能的并发队列。这也是为什么顺序队列比链式队列更常用的原因。
7、线程过多会造成资源利用效率过低。使用队列实现的拒绝策略有三种。
一种是直接拒绝，一种是先进先出，另外也可以使用阻塞队列。
使用链式队列实现的线程池叫无边界线程池，使用数组实现的队列叫有边界线程池。
无边界线程池，可以支持更多的队列，但是可能造成线程悬挂，等待。对于资源有限的，一般使用有边界队列。
常见的有：连接池、数据库连接、http连接
