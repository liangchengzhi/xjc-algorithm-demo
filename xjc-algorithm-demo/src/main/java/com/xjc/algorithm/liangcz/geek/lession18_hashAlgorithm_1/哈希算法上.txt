# 什么是哈希算法？
把任意长度的二进制转换为固定长度的二进制
哈希有几点要求：
1、不能反向推导
2、对原始数据非常敏感，只要修改一点，哈希值就不一样
3、散列冲突概率要尽量低
4、要尽量高效
哈希算法的应用包括：
安全加密、唯一标识、数据校验、散列函数、负载均衡、数据分片、分布式存储
# 安全加密
128位长度的哈希，因为任意数据都可以加密为哈希，所以根据鸽子理论，还是会有散列冲突
但是哈希算法冲突的概率还是很低的，特别是SHA、SHA-256、SHA-1，更加复杂，更加不可逆
# 唯一性
文件取哈希，确定唯一性
# 数据校验
BT 下载就是在把下载好的文件签名再种子文件中，防止文件被篡改
# 散列函数
hashmap的散列函数，是使用哈希算法生成
# 如何使用哈希算法加密保证安全
哈希算法本身可逆概率很低，但是有些密码比较简单，可以使用字典攻击来破解，因此使用“加盐”的方式可以避免破解
# 安全和攻击是一种博弈关系
安全只是增加破解成本，同时也会降低我们效率，越复杂的算法，越会降低我们的效率，因此在实际应用中药权衡
